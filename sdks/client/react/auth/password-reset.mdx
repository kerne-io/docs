---
title: "Password reset"
description: "Reset forgotten passwords"
---

The password reset flow allows users to recover access to their accounts. It works in two steps:

1. **Request reset** - User submits their email to receive a reset link
2. **Reset password** - User clicks the link and sets a new password

## Request password reset

Send a password reset email to the user.

### Signature

```typescript
await kerne.auth.requestPasswordReset(email: string): Promise<void>
```

### Example

```typescript
await kerne.auth.requestPasswordReset('user@example.com');
// Email sent with reset link
```

> [!NOTE]
> This method always succeeds, even if the email doesn't exist. This prevents email enumeration attacks.

---

## Reset password

Set a new password using the reset token from the email.

### Signature

```typescript
await kerne.auth.resetPassword(token: string, password: string): Promise<void>
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `token` | `string` | Yes | Reset token from the email link. |
| `password` | `string` | Yes | New password (min 8 chars). |

### Example

```typescript
// Token comes from URL query param
const token = new URLSearchParams(window.location.search).get('token');

await kerne.auth.resetPassword(token, 'newSecurePassword123');
```

---

## Possible errors

| Code | Status | Description |
|------|--------|-------------|
| `K1006` | 401 | **Token expired**: The reset token has expired. |
| `K1007` | 401 | **Invalid token**: The reset token is invalid. |
| `K2002` | 400 | **Weak password**: Password doesn't meet requirements. |
