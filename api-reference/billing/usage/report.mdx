---
title: "Report usage"
api: "POST /v1/admin/billing/usage"
description: "Report feature usage for a scope (user or project)"
---

## Authentication

<Note>
This endpoint requires **secret key** authentication (`sk_live_...`).
</Note>

## Request body

<ParamField body="scopeId" type="string" required>
  The ID of the user or project
</ParamField>

<ParamField body="featureKey" type="string" required>
  The key of the feature being used
</ParamField>

<ParamField body="amount" type="number" default={1}>
  The amount of units consumed
</ParamField>

<ParamField body="idempotencyKey" type="string">
  Optional key to prevent duplicate processing of the same event
</ParamField>

## Response

This endpoint returns `202 Accepted` immediately. Processing happens asynchronously.

## Errors

| Code | Status | Description |
|------|--------|-------------|
| `K1007` | 401 | **Unauthorized**: Invalid or missing secret key. |
| `K4002` | 404 | **Application not found**: The specified application slug is invalid. |
| `K3001` | 404 | **Feature not found**: The specified feature key does not exist. |
