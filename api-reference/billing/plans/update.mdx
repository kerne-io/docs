---
title: "Update plan"
api: "PATCH /v1/billing/plans/:id"
---

Update an existing plan.

## Authentication

<Note>
This endpoint requires **secret key** authentication (`sk_live_...`).
</Note>

## Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | `string` | Plan ID. |

## Request body

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | `string` | Plan name. |
| `description` | `string` | Plan description. |
| `is_active` | `boolean` | Whether the plan is active. |
| `features` | `array` | Updated feature entitlements. |

## Example request

```json
{
  "name": "Pro Monthly (Updated)",
  "is_active": false
}
```

## Response

```json
{
  "id": "plan_01234567890abcdef",
  "name": "Pro Monthly (Updated)",
  "is_active": false,
  "updated_at": "2025-01-02T00:00:00.000Z"
}
```

## Errors

| Code | Status | Description |
|------|--------|-------------|
| `K4001` | 404 | Plan not found. |
| `K2001` | 400 | Invalid request payload. |
| `K3002` | 403 | Insufficient permissions. |
