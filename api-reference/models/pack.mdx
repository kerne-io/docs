title: "Pack"
description: "Entitlement pack model"
---

A Pack is a versioned snapshot of entitlement definitions attached to a Plan. Packs enable grandfatheringâ€”existing subscribers keep their original entitlements when you create new versions.

## Properties

| Property | Type | Description |
|----------|------|-------------|
| `id` | string | Unique identifier for the pack |
| `plan_id` | string | Plan this pack belongs to |
| `version` | number | Version number (auto-incremented) |
| `status` | enum | `DRAFT`, `ACTIVE`, or `ARCHIVED` |
| `created_at` | string | ISO 8601 timestamp |
| `entitlements` | array | List of [PackEntitlement](#packentitlement) objects |

## Pack status

| Status | Description |
|--------|-------------|
| `DRAFT` | Pack is being configured, not yet usable |
| `ACTIVE` | Current pack for new subscriptions |
| `ARCHIVED` | Old pack, kept for grandfathered subscribers |

## PackEntitlement

| Property | Type | Description |
|----------|------|-------------|
| `id` | string | Unique identifier |
| `feature_id` | string | Reference to the Feature |
| `feature_key` | string | Feature key (for convenience) |
| `feature_name` | string | Feature name (for convenience) |
| `feature_type` | string | Feature type (for convenience) |
| `limit` | number | Numeric limit (nullable) |
| `value` | string | String value for static features (nullable) |

